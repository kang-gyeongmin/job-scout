# job-scout 설계 문서

날짜: 2026-07-09
상태: 사용자 설계 승인 완료 (구두), 스펙 리뷰 대기

## 목적

원티드·사람인·로켓펀치에서 **데이터 엔지니어(신입·주니어, 서울·수도권)** 채용 공고를 매일 수집하고, Claude Agent SDK 기반 에이전트가 사용자 프로필 대비 **적합도 점수(1~10)·이유·3줄 요약**을 생성해 **이메일로 발송**하는 개인용 알림 봇.

## 확정된 요구사항

| 항목 | 결정 |
|---|---|
| 결과물 형태 | 정기 알림 봇 |
| 알림 채널 | 이메일 (SMTP, 네이버 메일 계정 사용 가능) |
| 실행 환경 | 사용자 PC + Windows 작업 스케줄러 (매일 1회) |
| 기본 직무 | 데이터 엔지니어 (config로 변경 가능) |
| 기본 조건 | 신입~주니어(3년차 이하), 서울·수도권 |
| LLM 역할 | 수집은 코드가, 적합도 판단·요약은 에이전트가 (접근 방식 B: 하이브리드) |
| 인증 | Claude 구독 인증 또는 Console API 키(.env). 무료 계정은 불가 — 종량제 시 월 $0.2~1 예상 |

## 아키텍처 (접근 방식 B: 하이브리드)

수집기는 결정적인 Python 함수로 구현하고, Claude Agent SDK(`claude-agent-sdk` Python)의 **커스텀 툴**로 등록한다. 에이전트는 툴을 호출해 검색하고(필요 시 키워드를 바꿔 재검색), 프로필과 비교해 채점·요약한다. 이메일 발송 등 외부로 나가는 동작은 에이전트가 아닌 코드가 수행한다.

기각한 대안:
- A. 순수 파이프라인(Agent SDK 없이 API 직접 호출) — 프로젝트 취지(헤르네스 학습)와 불일치
- C. 풀 에이전트(Playwright 브라우징 수집) — 매일 실행하기엔 느리고 비용·불안정성 큼

## 프로젝트 구조

```
job-scout/
├─ config.yaml          # 검색 키워드, 경력, 지역, 사이트 on/off, 수신 이메일
├─ profile.md           # 사용자 프로필·기술스택 요약 (적합도 판단 기준)
├─ src/
│  ├─ collectors/       # 사이트별 수집기 — 공통 JobPosting 스키마 반환
│  │  ├─ wanted.py      #   원티드: 비공식 JSON API (1단계 MVP)
│  │  ├─ saramin.py     #   사람인: 공식 오픈 API, 무료 키 발급 신청 필요 (2단계)
│  │  └─ rocketpunch.py #   로켓펀치: HTML 크롤링, 가장 깨지기 쉬움 (3단계)
│  ├─ agent.py          # Agent SDK: 커스텀 툴 등록, 채점·요약, 구조화된 JSON 출력
│  ├─ mailer.py         # SMTP HTML 이메일 발송
│  ├─ store.py          # seen.json — 발송 완료 공고 ID 기록 (중복 알림 방지)
│  └─ main.py           # 진입점: 에이전트 실행(툴로 수집 주도) → 결과 이메일 발송
├─ tests/               # 수집기 단위 테스트 (저장된 응답 fixture 기반)
├─ reports/             # 이메일 실패 시 로컬 백업 리포트 (gitignore)
├─ logs/                # 실행 로그 (gitignore)
├─ .env                 # ANTHROPIC_API_KEY, SARAMIN_API_KEY, SMTP 계정 (gitignore)
└─ scripts/register_task.ps1  # 작업 스케줄러 등록
```

### 공통 데이터 스키마

`JobPosting`: `id`(사이트 접두어 포함 고유 ID), `site`, `title`, `company`, `location`, `experience`, `url`, `description`, `posted_at`

## 동작 흐름

1. 작업 스케줄러가 매일 정해진 시간에 `main.py` 실행
2. 에이전트에 미션 전달: profile.md + config 조건에 맞는 공고 탐색
3. 에이전트가 `search_jobs(site, keyword)` 툴 호출 — 키워드 변형("데이터 엔지니어", "Data Engineer", "ETL" 등) 재검색 가능
4. 툴 계층에서 `seen.json` 등재 공고 자동 제외 → 신규 공고만 반환
5. 에이전트가 공고별 적합도 점수(1~10)·이유·3줄 요약을 구조화된 JSON으로 출력
6. 코드가 점수순 정렬 → HTML 이메일 발송(공고 링크 포함) → `seen.json` 갱신

## 에러 처리

- 사이트 수집 실패: 해당 사이트만 건너뛰고 이메일 하단에 실패 사실 명시
- 이메일 발송 실패: `reports/YYYY-MM-DD.md`로 로컬 저장해 유실 방지
- 신규 공고 0건: 이메일 미발송 (config 옵션으로 "오늘은 없음" 알림 선택 가능)
- 실행 로그를 `logs/`에 기록해 스케줄러 동작 여부 확인 가능

## 크롤링 원칙

개인 용도로만 사용. 요청 간격(rate limit)을 두고 하루 1회 소량만 수집. 로그인 우회·대량 수집 없음.

## 테스트 전략

- 수집기: 실제 응답을 fixture로 저장해 파싱 로직 단위 테스트 (네트워크 없이 실행)
- 에이전트: `--dry-run` 모드(이메일 미발송, 콘솔 출력)로 프롬프트 품질 확인
- 이메일: dry-run 검증 후 본인 계정 1회 실발송 확인

## 구현 단계

1. **MVP**: 원티드 수집기 + 에이전트 채점 + 이메일 발송 + dry-run
2. 사람인 추가 (API 키 발급 후)
3. 로켓펀치 추가
4. 작업 스케줄러 등록 스크립트 + 운영 문서
